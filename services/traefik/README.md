# Traefik Service Module

This module deploys Traefik as the ingress service.

## Deployment model

- Runtime compose file is generated by the Nix module and written to:
  - `/etc/traefik/docker-compose.yml`
- Generation is required because Traefik compose content depends on module options
  (for example metrics entrypoint flags, HTTP->HTTPS redirect flags, and TLS wiring).
- systemd runs `docker compose up -d` / `docker compose down` from `/etc/traefik`.

## Source files in repo

- `services/traefik/traefik.nix`: module wiring and systemd service.
- `services/traefik/options.nix`: module options.
- `services/traefik/render.nix`: compose/tls rendered text.
- `services/traefik/docker-compose.yml`: checked-in reference snapshot (not the
  active runtime source).

## Why Traefik differs from most services

Most services in this repository use checked-in compose files and generate
service-specific config files. Traefik is different because ingress behavior is
more option-driven at the compose layer.
